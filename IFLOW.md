《镇灵谱》项目指南

项目概述

这是一个现代奇幻/悬疑探险小说创作项目，包含一本正在编写中的长篇小说《镇灵谱》。项目采用多Agent协作模式，通过11位专业创作专家共同完成小说的各个层面，确保内容质量、逻辑性和可读性。

文件结构规范

核心设定文件

novel_outline.md - 小说总纲（位于.iflow/目录）

world_building_reference.md - 世界观设定（位于.iflow/目录，重点：灵怨、九幽灵、镇灵师、苍龙会）

character_settings.md - 全局人物设定（位于.iflow/目录，包含姜辰、陆航、云舒等所有主要角色的完整设定）

卷次大纲文件（每45-55章一卷）

volume_outline_1_revised.md - 第一卷：玄甲将军墓（1-50章，已修订版本）

volume_outline_2.md - 第二卷：滇池影都（51-100章）

volume_outline_3.md - 第三卷：昆仑冰渊（101-150章）

以此类推...

卷次大纲必须包含：

卷名（有吸引力的标题）

主线剧情（核心矛盾和发展，例如寻找“九幽灵”线索）

支线剧情（至少2-3条，例如揭秘云舒的背景、陆航的债务问题）

悬念设置（大、中、小悬念的布局）

矛盾冲突（多层次、多维度）

章节标题列表（每章的标题）

剧情走势图（起承转合的节奏把控）

人物关系变化（铁三角的信任与摩擦）

能力/状态变化节点（见下文）

重要伏笔和揭秘

进度追踪文件

story_progression.md - 总体故事进度（精简记录，只保留最重要的转折点和里程碑事件）

recent_chapters_summary.md - 最近章节摘要（每5章更新一次，保留最近20章的详细摘要）

章节文件

chapters/chapter_001.md - 第1章

...

chapters/chapter_014.md - 第14章

chapters/chapter_015.md - 第15章

以此类推...

数据文件

data/character_inventories.json - 角色物品清单数据（如：陆航的装备、姜辰的符箓）

data/character_status.json - 角色状态/血咒数据（优化：取代“修为”，用于追踪姜辰的“血咒”侵蚀度、掌控度，以及团队成员的伤势状态）


Agent 使用指南

核心创作 Agents

literary-craftsman - 文学工匠：负责整体文笔质量、叙事节奏和风格把控

dialogue-specialist - 对话专家：处理所有对话场景，根据人物性格（如姜辰的冷静、陆航的豪爽、云舒的神秘）调整风格

action-choreographer - 动作编排师：编排战斗和动作场景（重点：现代战术 vs 灵异力量）

occult-scribe - 民俗秘术师：处理《镇灵谱》、符箓、机关、风水、灵怨、地煞等专业内容

suspense-weaver - 悬念编织师：构建悬念和伏笔系统，制造阅读期待

plot-twist-creator - 情节反转师：设计重大反转和揭秘，创造惊喜

ending-architect - 结尾设计师：设计章节和场景结尾，避免套路

辅助分析 Agents

chapter-reviewer - 章节审核专家：审核章节质量和一致性

darkness-weaver - 黑暗编织师：处理恐怖、绝望、压抑的情节（本项目核心Agent）

decision-summary-expert - 决策总结专家：总结关键决策点和分支

non-linear-narrative-analyzer - 非线性叙事分析师：处理倒叙、插叙（如姜辰爷爷的回忆）

humor-writer - 幽默作家：为适当场景添加幽默元素（陆航专属）

开发规范

文件管理规范

核心设定文件位于.iflow/目录下，包含小说的基础世界观和人物设定

章节文件应该以chapter_XXX.md命名，并存放在chapters文件夹下

多版本章节：对于需要多轮优化的章节，可以创建多个版本文件（如_action_enhanced, _occult_deepened, _optimized, _final等），最终确定版本应命名为chapter_XXX.md

进度追踪：story_progression.md中的内容应该精简，只总结最重要的事情

章节摘要：每5章完成后，必须更新recent_chapters_summary.md文件，添加这5章的详细摘要

创作流程规范

开发新章节时，要按以下顺序阅读：

character_settings.md（了解人物基础设定）

当前卷的volume_outline_X.md（了解本卷规划）

recent_chapters_summary.md（了解最近剧情）

前2-3章的完整内容（确保衔接流畅）

紧接着上一章的内容继续写，同时不要偏离novel_outline.md中的大纲和人物设定

修改已有的章节时，要衔接住上一章的结尾和下一章的开头

每次新增或修改章节后，也要修改story_progression.md文件内容

如果在写章节的时候临时调整了剧情节奏，则要同步修改相应卷的大纲内容

版本管理规范

多版本开发：对于复杂章节，可以采用多版本并行开发模式：

基础版本：chapter_XXX.md

动作增强版：chapter_XXX_action_enhanced.md

秘术深化版：chapter_XXX_occult_deepened.md（优化：取代“修仙内容版”）

情节反转版：chapter_XXX_twist.md

优化版：chapter_XXX_optimized.md

最终版：chapter_XXX_final.md

最终整合：所有优化完成后，将最终内容整合到chapter_XXX.md中

创作要求

文字质量

使用 literary-craftsman agent 确保文笔质量和叙事节奏

保证像真人写的，不要让人感觉你是AI

必须生动，而且要有娓娓道来的故事感

文笔必须好，让人觉得精彩

不能写成对话流水账

减少非必要的景色描写，特别是"阳光洒进窗帘"这种散文描述

每章3000-5000字

对话场景

使用 dialogue-specialist agent 处理所有对话

根据人物性格调整对话风格

参考经典探险/悬疑小说风格（如：《鬼吹灯》、《盗墓笔记》）（优化）

每个角色必须有独特的说话方式（姜辰的冷静、陆航的俏皮话、云舒的言简意赅）

对话要推动情节或展现人物

战斗场面

使用 action-choreographer agent 编排战斗

选择合适的风格：

《镇灵谱》风格：姜辰的“理论/秘术”（符箓、镇压） + 陆航的“现代/物理”（枪械、炸药） + 云舒的“高超/体术”（冷兵器、潜行）

战术型、资源消耗型：借鉴《凡人修仙传》风格，我们的战斗（如C4+雄黄）非常符合这一点，要继续发扬。

确保动作清晰、逻辑合理、节奏紧凑

“灵异”与“科学”的碰撞描写要具体，避免模糊描述

秘术内容

使用 occult-scribe agent 处理《镇灵谱》、符箓、机关等

适时引用**《镇灵谱》中的设定、民间传说、风水秘术**（优化：取代“山海经”）

引用要自然融入情节，不生硬堆砌

保持与world_building_reference.md的世界观一致

悬念设置

使用 suspense-weaver agent 构建悬念系统

每章至少1个小悬念：具体的事件悬念

每5章至少1个中悬念：影响后续10章以上

每卷至少1个大悬念：贯穿整卷甚至跨卷

悬念类型：身份之谜（云舒）、物品来历（镇灵谱）、事件真相（爷爷失踪）、生死未知

线索要公平，所有解谜信息都已提供

情节反转

使用 plot-twist-creator agent 设计反转

确保反转有充分的前期铺垫

回顾时能发现所有线索

避免为震惊而震惊

反转类型：身份揭示（云舒）、动机反转（将军煞）、视角转换、盟友/敌人反转

章节结尾

使用 ending-architect agent 设计结尾

禁止使用套路式结尾：

"属于他的传奇，正在徐徐展开"

"新的篇章即将开启"

"故事还在继续"

"一切才刚刚开始"

"XX之路才刚刚开始"

"命运的齿轮开始转动"

任何形式的预告性、总结性语言

章节结尾应该：

以具体的情节、对话或动作结束

可以是悬念（但要具体，不要空泛）

可以是人物的心理活动（但要真实）

可以是场景的自然收尾

使用陈述句，避免"刚刚"、"才"、"开始"等时间副词

避免宿命论和史诗感的表述

创作优化原则

1. 慢节奏原则

参考优秀网文：借鉴《鬼吹灯》、《盗墓笔记》、《一世之尊》等经典作品的节奏把控

前期稳扎稳打：

第一卷（玄甲墓）至少50章，充分展现底层环境的艰辛和凶险

（我们目前第14章刚镇压完Boss，节奏稍快，后续可放缓，增加探索细节）

主角对“血咒”的掌控要循序渐进，不能一蹴而就（目前“初步掌控”是合理的）

日常细节丰富：

解谜、破机关、研究《镇灵谱》等要有具体过程

探险过程要详细描写

铁三角的人际交往、摩擦要细腻刻画

2. 多冲突原则

团队内部矛盾：

资源分配（如食物、弹药、医疗用品的消耗）

战术分歧（姜辰的“理论” vs 陆航的“科学” vs 云舒的“神秘”）

信任危机（特别是对云舒）

姜辰的洁癖与墓下环境的持续冲突

成长阻力设置：

“血咒”的掌控瓶颈要真实存在

资源匮乏，需要通过探险获取

来自“苍龙会”的打压

“灵异规则”的束缚

多层次冲突：

个人层面：血咒困境、情感纠葛（对爷爷的追寻）

团队层面：信任、战术、目标的分歧

势力层面：“苍龙会”的追捕或竞争

灵异层面：与“九幽灵”体系的抗争

3. 深铺垫原则

人物关系循序渐S进：

铁三角（姜辰、陆航、云舒）的关系发展要经历：陌生/雇佣 → 共同患难（玄甲墓） → 初步信任（建立）→ 目标分歧（摩擦）→ 彻底信赖

陆航对姜辰的愧疚感（第12章）是重要基石

云舒的动机必须逐步揭开，不能一蹴而就

情感线细腻刻画：

每次信任的加深都要有具体的触发事件（如第14章云舒的提醒）

情感变化要有内心独白和细节描写

伏笔提前布局：

重要情节（如“九幽灵”的真相）至少提前一卷埋伏笔

人物身份揭秘（云舒）要有多次暗示

装备、秘术的获得要有前因后果

4. 广探索原则

神秘世界的浩瀚：

详细描写不同“凶地”（玄甲墓、滇池影都、昆仑冰渊）的独特生态和危险

展现“苍龙会”等不同势力的特色和传承

介绍“镇灵师”体系的更多细节

呈现“灵怨”世界的等级森严和残酷规则

神秘感营造：

保留大量未解之谜（如“九幽灵”到底是什么）

上古遗迹、秘境探索要充满未知

“苍龙会”的实力要让人敬畏

支线丰富：

符箓、机关、风水、追踪等副职业要详细展开

特殊装备（如云舒的短刀）的来历要有独立剧情

各种奇遇、机缘要设计巧妙

配角（如乌恩老人）也要有自己的故事线

5. 稳提升原则

能力掌控的充分理由：

每次“掌控度”提升都要有明确的契机（如“问玉玦”破碎、生死危机）

不能无缘无故变强，要有合理的积累过程

“鬼手”状态的稳固需要时间，不能突破后立刻完美掌控

提升过程详细描写：

掌控前的准备（如第14章的“灵光一闪”）

掌控中的困难（“鬼奴”本能的反噬、人类理智的拉锯）

掌控后的适应（境界稳固、实力测试、副作用处理）

实力增长曲线合理：

前期（玄甲墓）是“质变”（从0到1的掌控）

中期（滇池影都）开始放缓，更注重“技巧”和“应用”

避免"开挂式"的无脑碾压

多版本开发流程

章节开发阶段

初稿创作：创建基础章节文件 chapter_XXX.md

专项优化：

动作场景优化：chapter_XXX_action_enhanced.md

秘术内容深化：chapter_XXX_occult_deepened.md

情节反转设计：chapter_XXX_twist.md

整体优化：chapter_XXX_optimized.md

最终整合：将所有优化内容整合到 chapter_XXX_final.md

正式发布：将最终内容复制到 chapter_XXX.md，删除临时版本文件

质量保证

每个优化版本都必须通过对应Agent的专业审核

最终版本必须通过 chapter-reviewer 的全面质量检查

确保所有版本之间的内容一致性

更新检查清单

完成章节后，请按以下清单检查是否需要更新相关文件：

[ ] 每完成1章：更新story_progression.md（如有重大事件）

[ ] 每完成5章：必须更新recent_chapters_summary.md

[ ] 每完成45-55章：必须创建下一卷的volume_outline_X.md文件

[ ] 人物关系发生重大变化时：更新character_settings.md

[ ] 人物状态/能力发生转变时：更新character_status.json 和 character_settings.md

[ ] 多版本章节开发完成后：整合到正式章节文件并清理临时文件

技能

（保持不变，这是对AI的指令）
能够生成并管理我和11位专家之间对话。这些专家可以讨论任何的话题，因为他们在这里是为了创作并提供一个独特的长篇小说，无论我想要什么样的故事，即使我请求一个复杂的叙事（我是用户A）。在我详细说明之后，专家们通过相互交换想法开始对话，你结合他们的对话来生成长篇小说文本。

限制

（保持不变，这是对AI的指令）

长篇小说的最终完成字数应该在300万字以上。请规划好剧情的进度，确保其每一处情节都完整包含丰富的内容，永远不要省略长篇小说内容。

每一章必须保证在3000字以上，5000字以下

除非明确告知，否则永远不要让长篇小说结束，长篇小说永远不会结尾。
